- [x] Session validation: `--validate-session URL` headlessly checks whether saved session is still live (detects login walls vs real content)

- [x] Book-specific adapter maps for UnderDog, DraftKings, FanDuel (CSS selector hints + extra wait durations)

- [x] `test_backtest.py` rewritten as proper mocked unit tests (no live API calls at import/runtime; 65/65 tests pass offline)
- [x] New tests: `test_match_book_domain`, `test_validate_session_detects_login_wall`, `test_validate_session_accepts_content_rich_page`

### Auth workflow (manual then automatic)
1. **Manual (one-time)**: `python3 -m nba_model.model.web_text_ingestion --login https://app.underdogfantasy.com/pick-em --browser-auth-state-file data/config/auth/underdog_state.json`
   - A visible browser opens. You log in manually. Close the browser when done. Session cookies saved.
2. **Automatic (every run)**: `python3 -m nba_model.data.daily_etl --browser-auth-state-file data/config/auth/underdog_state.json --web-text-urls-file data/config/web_text_urls.txt ...`
   - Headless browser loads saved session and fetches content as an authenticated user.
3. **Re-login**: When sessions expire, re-run step 1. Use `--validate-session` to check first.

- [ ] Convert remaining manual helper scripts (`check_data.py`, `test_database.py`) into formal tests or move to a scripts folder

### Remaining limitations

- [ ] Parser recall depends on whether the saved session state actually reproduces cookie/token state in headless mode (some sites invalidate sessions across browser fingerprints)

## 5) Known Gaps / Constraints

- Reverse-engineering quality depends on enough overlapping `betting_lines` + model predictions in the same date/player/stat windows.
- Name mismatches across sources can still reduce join coverage (e.g., punctuation/suffix variants in player names).
- Some evaluation scripts still produce empty artifacts when windows are sparse; they now fail less often, but output quality remains data-dependent.
- ETL scheduler exists (GitHub Actions), but external alerting/monitoring on failed or partial runs is still minimal.
- UnderDog anonymous access returns login-gated UI; use `--login` to save session first, then headless fetch with `--browser-auth-state-file`.
- Session persistence depends on browser fingerprint compatibility; some sites may invalidate headless sessions saved from headed logins.
- Authenticated browser ingestion requires project `.venv` + Playwright browser binaries (`.playwright-browsers`).
- CSS selector hints for book-specific waits are best-effort; DOM changes on sportsbook sites will need adapter updates.

## 6) Quick Runbook Commands

```bash
# Daily ETL (strict + expand player pool + skip known zero-game players)
python3 -m nba_model.data.daily_etl \
  --strict \
  --all-db-players \
  --min-players 40 \
  --skip-zero-game-players \
  --odds-min-hours-between-polls 24

# Daily ETL (single pass reverse-engineering only)
python3 -m nba_model.data.daily_etl \
  --strict \
  --reverse-engineering-single-pass

# Daily ETL (direct website text snapshot fetch, once/day window)
python3 -m nba_model.data.daily_etl \
  --skip-game-logs \
  --skip-team-defense \
  --skip-odds \
  --skip-reverse-engineering \
  --web-text-urls "https://example.com/sportsbook/nba/props" \
  --web-text-min-hours-between-polls 24

# Step 1: Manual login (one-time per session expiry)
# Opens a VISIBLE browser window -- log in, then close the window.
python3 -m nba_model.model.web_text_ingestion \
  --login https://app.underdogfantasy.com/pick-em/higher-lower/all/NBA \
  --browser-auth-state-file data/config/auth/underdog_state.json \
  --browser-user-data-dir data/config/auth/underdog_profile

# Step 2: Validate saved session still works (optional)
python3 -m nba_model.model.web_text_ingestion \
  --validate-session https://app.underdogfantasy.com/pick-em/higher-lower/all/NBA \
  --browser-auth-state-file data/config/auth/underdog_state.json

# Step 3: Headless fetch using saved session
python3 -m nba_model.model.web_text_ingestion \
  --urls-file data/config/web_text_urls.txt \
  --browser-auth-state-file data/config/auth/underdog_state.json \
  --force-poll

# Daily ETL with login-before-run (interactive login then auto-ETL)
python3 -m nba_model.data.daily_etl \
  --skip-game-logs --skip-team-defense --skip-odds --skip-reverse-engineering \
  --web-text-urls-file data/config/web_text_urls.txt \
  --login https://app.underdogfantasy.com/pick-em/higher-lower/all/NBA \
  --browser-auth-state-file data/config/auth/underdog_state.json

# Daily ETL with session validation before web-text ingestion
python3 -m nba_model.data.daily_etl \
  --skip-game-logs --skip-team-defense --skip-odds --skip-reverse-engineering \
  --web-text-urls-file data/config/web_text_urls.txt \
  --browser-auth-state-file data/config/auth/underdog_state.json \
  --validate-session-before-etl

# Standalone browser parser over stored snapshots
python3 -m nba_model.model.browser_prop_parser \
  --urls-file data/config/web_text_urls.txt \
  --min-parse-confidence 0.50 \
  --max-snapshots-per-url 2

# Sync active NBA player reference list (for filtering non-NBA names)
python3 -m nba_model.model.web_text_ingestion \
  --sync-active-players-ref \
  --active-players-output-file data/config/active_nba_players.txt

# Daily ETL (continuous reverse-engineering readiness loop)
python3 -m nba_model.data.daily_etl \
  --strict \
  --reverse-engineering-poll-seconds 300 \
  --reverse-engineering-min-inferred-rows 25 \
  --reverse-engineering-min-book-stat-groups 2 \
  --reverse-engineering-min-player-segment-groups 5 \
  --reverse-engineering-require-stability-runs 2 \
  --reverse-engineering-stability-tolerance 0.10

# Real-data benchmark (player/window CI exports)
python3 -m nba_model.evaluation.run_real_data_benchmark \
  --windows 5 7 10 15 \
  --stat-types points \
  --distributions normal \
  --start-date 2024-11-01 \
  --end-date 2025-03-15

# Distribution sweep
python3 -m nba_model.evaluation.run_distribution_sweep \
  --windows 5 7 10 15 \
  --stat-types points assists rebounds pra \
  --start-date 2024-11-01 \
  --end-date 2025-03-15

# Line comparison
python3 -m nba_model.evaluation.line_comparison \
  --start-date 2024-11-01 \
  --end-date 2025-03-15 \
  --stat-types points assists rebounds pra \
  --edge-threshold 0.02

# Monthly diagnostics
python3 -m nba_model.evaluation.monthly_diagnostics \
  --start-date 2024-11-01 \
  --end-date 2025-03-15 \
  --stat-types points assists rebounds pra

# Reverse-engineering direct run (outside ETL)
python3 -m nba_model.evaluation.market_reverse_engineering \
  --continuous \
  --source both \
  --poll-seconds 300 \
  --min-inferred-rows 25 \
  --min-book-stat-groups 2 \
  --min-player-segment-groups 5 \
  --require-stability-runs 2 \
  --stability-tolerance 0.10
```


================================================================================
TECHNICAL DEBT / KNOWN ISSUES
================================================================================

- Daily ETL runner exists and is scheduled, but external monitoring/alerting is still minimal.
- Reverse-engineering outputs are more stable with continuous mode, but remain data-coverage constrained.
- Line comparison/monthly diagnostics can still return sparse/empty artifacts when `betting_lines` and `predictions` do not overlap enough.
- Real-data benchmark/distribution sweep scripts are implemented; production defaults still need formal selection from full-run artifacts.
- UnderDog fetch requires authenticated session; use `--login` to manually save session, then `--browser-auth-state-file` for headless reuse.
- Session persistence across headed->headless may fail if site validates browser fingerprint (user-agent, TLS, etc).
- CSS selector wait hints for books are best-effort; DOM restructures require updating `_BOOK_WAIT_SELECTORS` map.
- Current travel/rest/injury features are heuristic; no independent validation source is integrated yet.
- Some modules still use `print()` instead of structured logging.
- DB path/config remains mostly default-arg based; centralized env config is incomplete.
- No explicit handling for player trades/team-context shifts mid-season.
