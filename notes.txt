## Near-term objective

1. **Book Edge Scanner** — the marquee analysis surface. Pick books, rank the
   whole slate's props by model edge (fitted-normal μ vs the book line; e.g.
   Underdog 17.5 vs model μ=19 → P(over) > 50%). Keep expanding it.
2. **Player Charts** — the per-player deep-dive surface. Charts/splits/
   decisional tools are broad now; remaining work is mostly Tk parity.
3. **Cross-book consensus** — activate more of the 11 stub scrapers (9 of 20
   produce data) so consensus / edge math has more books behind it.
   **(Blocked on the live-Chrome host — see below.)**
4. **Team-line priors** feed the player model (pace + implied team total) so
   the chart book-mean and the model μ share one cross-book signal. (Wired.)
5. **Multi-sport** — NFL first. Schema + resolution + validation scaffolding
   is in place; NFL ingestion + scrapers are the next big lift. Do NOT ship
   non-NBA data yet.

## Current architecture (high level)

- **Python 3.11**, SQLite at `data/database/nba_data.db`, pytest suite.
- **Data ingest**: `nba_model/data/nba_results_ingestion.py` (bulk team-game +
  player-game-logs from nba_api), `daily_etl.py` (orchestrated daily job),
  `hourly_update.py` (hourly scrape + recompute on the dev-Mac Chrome host).
- **Scrapers**: `nba_model/scrapers/` per-book registry (20 books, 9 with
  parsers). Web fetch via Playwright + CDP through real Chrome on `:9222`
  (`web_text_ingestion.py`); parsers in `browser_prop_parser.py` (props) and
  `team_line_parser.py` (team lines).
- **Model**: `simulation.py` (8 distributions + `blend_team_prior`),
  `prop_board.py`, `probability.py`, `minutes_projection.py`,
  `defense_adjustment.py`, `correlation_calibration.py`, `edge_scanner.py`.
- **Visualization**: `player_charts.py` (matplotlib + pure data helpers, used
  by Tk) and `plotly_charts.py` (interactive web equivalents).
- **Web**: Streamlit `nba_model/web/app.py` + auth/billing (`auth.py`,
  `subscriptions.py`, `stripe_helpers.py`, `webhook_app.py`, `throttle.py`).
- **Desktop**: Tk `nba_model/simple_ui.py`.
- **Multi-sport**: `sports/` package (nba live; nfl/mlb/nhl/soccer stubs).

Constraints (do not violate):
- Browser/ETL always via `.venv/bin/python3` (never system python3).
- PrizePicks/Underdog scraping needs real Chrome on `:9222` (CDP); not CI.
- Operations panel + manual-lines DB save + Admin dashboard are admin-gated
  regardless of `BILLING_ENABLED`.
- All user inputs go through `nba_model/web/input_validation.py`;
  plausible-line check at DB insert.
- Tk = matplotlib; web = Plotly (no `st.pyplot` in web views).
- Do not delete `sports/` scaffolding.

## Implemented (trust unless a regression is found)

### Book Edge Scanner
- `nba_model/model/edge_scanner.py` — `fetch_latest_prop_lines` (deduped
  latest line per player/stat/book from `web_prop_cards`, active_nba only,
  48h lookback), `score_prop_edges` (fitted-normal μ/σ per player+stat,
  memoized; P(over), edge vs implied, EV; DFS lines priced at the -110
  breakeven), `top_edges` (filter + rank by `model_edge`). CLI:
  `python -m nba_model.model.edge_scanner --books underdog prizepicks ...`.
- Streamlit **"Line edge scanner"** view: book multiselect, stat / window /
  min-edge / min-P(over) filters, model-mode toggle (chart_mean | rolling),
  KPI row, ProgressColumn table, CSV export, "open in Player Charts" jump.
  Free tier gated to preview players/stats + 8 scans/min/session throttle.

### Model
- 8 distribution families in `simulation.py`; `negative_binomial` reconciled
  with chart overlays via `normalize_distribution_name`.
- `DEFAULT_DISTRIBUTION_BY_STAT` calibrated (rebounds → poisson; others →
  normal pending market-line backfill).
- **Team-prior blender** `blend_team_prior()` wired into the live path:
  `run_single_prop` (`player_team`/`opponent_team`), `prop_board`, and the
  hourly recompute. DB seam: `get_team_prior_inputs` / `get_team_prior_inputs_map`.
  Baseline is per-team recent points (`get_team_recent_avg_points`) to match
  the per-team implied total (NOT the game total — that bug was fixed).
- `prob_over` (closed-form) handles σ≤0 correctly (1.0/0.5/0.0, not flat 0.0).
- Backtest computes `prob_over` at the resolved settlement line (decision /
  EV / Brier are now consistent in market/model-line modes).

### Player Charts (matplotlib for Tk + Plotly for web)
- Recent games, distribution (+best-EV highlights + line-ladder toggle),
  hit-rate, rolling-CI, trend/form, defense scatter, multi-player compare.
- Box/quantile, calendar heatmap (weekday × month), win/loss split,
  starter/bench split (minutes ≥ 20), **vs-opponent split**, **pace+minutes**,
  **cumulative line-vs-actual ribbon**, **CLV proxy**, **model-EV vs
  fitted-EV**, **parlay correlation heatmap**, **Kelly stake** (in the
  custom-line probe). Web Splits / Line-Movement / Hit-Rate tabs surface them.
- **Line Movement tab**: open→current drift per book from
  `betting_line_snapshots` (`fetch_line_movement_snapshots`).
- **Upcoming-game line selection**: `_fetch_latest_book_lines` prefers the
  player's next (today/upcoming) game's line, not `max(game_date)` / yesterday.
- Staleness label uses the real scrape timestamp (`scraped_at_utc`), not the
  game date.

### Web app / billing
- Feature parity with desktop `simple_ui.py` (Single Prop, Manual Lines admin
  save, Operations admin console, All-stats, Parlay, Compare).
- All NBA web charts on Plotly. URL query-string state, watchlist, ETL status,
  CSV downloads, Docker + `docker-compose.yml`.
- **Anonymous watchlist persistence** across sessions via a first-party cookie
  token (`anon:<token>`, hex-validated against cookie-injection); signed-out-
  under-billing stays session-only. Cap 25.
- **Stripe Customer Portal** "Manage subscription" link for premium users
  (`stripe_helpers.customer_portal_url`, env `STRIPE_PORTAL_RETURN_URL`).
- **Admin dashboard** ("Admin (dashboard)" view, admin-gated): premium/free/
  total counts, MRR estimate (`STRIPE_PRICE_MONTHLY_USD`), recent churn —
  `subscriptions.aggregate_stats`.
- **STRIPE_MODE=test|live** env toggle with mode-scoped keys.
- Webhook alert on `invoice.payment_failed` (`webhook_app.alert_payment_failed`,
  env `BILLING_ALERT_WEBHOOK_URL`).
- Free-tier app-layer scrape throttle (`web/throttle.py`).
- 7-day trial: pure `auth.trial_active` helper + `ENABLE_TRIAL`/`TRIAL_DAYS`
  (wiring into live tier resolution needs a `created_at` column — documented).
- `BILLING_ENABLED=0` open-access mode; billing dormant behind the flag.
- Webhook hardening (HMAC + replay tolerance + idempotency + 256 KiB cap +
  rate limit + slowloris timeout + OWASP headers) per docs/SECURITY.md.

### Data delivery / ops
- **Nightly DB publish hook**: `nba_model/data/publish_db.py` +
  `scripts/publish_db.sh` — backs up the DB, refuses to publish a locked DB,
  logs row counts, `--dry-run`, force-adds the gitignored DB, commits + pushes
  only on change (DEPLOYMENT.md §14).
- **ETL alerting**: `nba_model/data/etl_alerts.py` (`build_alert` +
  `maybe_send_alert`). Daily + hourly ETL embed an `alert` marker in the
  report JSON and accept `--alert-webhook-url`.
- **Predictions widened**: hourly recompute persists board lines to
  `predictions` for points/assists/rebounds/pra, idempotent per slate
  (`hourly_update._persist_predictions`).
- Hourly pipeline (`hourly_update.py`, `scripts/scheduler/*`); the fcntl lock
  has a Windows `msvcrt` fallback so the module imports cross-platform.

### Evaluation
- `run_distribution_sweep.py` settles each stat at a realistic per-stat line by
  default (`_resolve_stat_lines` + canonical `DEFAULT_LINES_BY_STAT`, shared
  with `run_per_stat_sweep`) — no more `line == mean` degeneracy unless
  `--line` / `--use-market-lines` is passed.
- `get_backtest_data` joins predictions↔logs with `DATE()` normalization.

### Multi-sport scaffolding
- `sport TEXT NOT NULL DEFAULT 'nba'` idempotent migration on `game_logs`,
  `betting_lines`, `betting_line_snapshots`, `players`, `web_prop_cards`,
  `predictions` (`DatabaseManager._ensure_sport_columns`, runs on init,
  upgrades legacy DBs in place).
- Per-`(book, sport)` scraper resolution (`BookScraper.sport`,
  `get_scraper_for_book_sport`, `get_scraper_for_url(..., sport=)`).
- `input_validation.validate_stat_type(..., sport=)` reads non-NBA stat lists
  from the `sports/` registry.
- `sports/` package: nba (live) + nfl/mlb/nhl/soccer stubs; Streamlit
  sport-picker shows roadmap cards for non-live sports.

### Scrapers / consensus
- `data/config/web_text_urls.txt`: 9 active books (PrizePicks, Underdog,
  Pick6, ParlayPlay, BetMGM, Caesars, DK, Bovada, Kalshi).
- Shared player-name resolver `scrapers/player_names.py`. CDP browser
  connections are now closed after each fetch (was leaking per URL).
- `kalshi._decimal_to_american` guards decimal ≤ 1.0 (no /0 dropping the
  whole snapshot); `prop_board._fetch_betting_lines_for_game` uses positional
  `IN (...)` placeholders (the named-tuple binding raised at runtime before).

## Known limitations / carry-forward
- 11 of 20 scrapers still parser-less (open authenticated-snapshot work).
- `web_prop_cards` / `betting_lines` / `betting_line_snapshots` are sparse in
  the current DB, so the Edge Scanner, Line Movement, ribbon, and CLV views
  show empty-states until the Chrome-host ETL populates them.
- Team-chart book reference is `points`-only (no team_total markets at lobby).
- `betting_lines` market coverage is ~2 days — blocks `--use-market-lines`.
- `players.team` is sparse; chart code falls back to `nba_active_players_ref`
  + `game_logs.matchup`.
- This dev box is Windows; the production scraping host is the dev Mac. Some
  tests use Windows-safe teardowns (`ignore_cleanup_errors`, explicit sqlite
  `close()`); `DataLoader` leaks its sqlite connection (pre-existing) which
  only matters for temp-dir cleanup on Windows.
- Tk `simple_ui.py` has not been updated with the newest chart builders.
