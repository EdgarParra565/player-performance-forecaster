save; web = Plotly, Tk = matplotlib; do not delete `sports/` scaffolding.

## Implemented (trust unless regression)

- **Edge Scanner** — `edge_scanner.py` + "Line edge scanner" Streamlit view; CLI
  `--books underdog ...`. Three modes (`MODEL_MODES`): `chart_mean` (default quick
  screen: last-N normal), `rolling`, and **`full`**. Output keeps `SCORED_COLUMNS`
  as ordered prefix + appends `distribution`/`model_mode` (`SCORED_COLUMNS_FULL`).
  DFS books assume -110. Unfittable rows are dropped, never raised.
- **WS9 unify scoring (2026-07-16)** — `full` mode runs the prop_board stack
  (rolling μ/σ + `blend_team_prior` + `get_default_distribution(stat)`), DB-direct
  (no DataLoader/network). Scanner full mode and hourly `predictions` share
  `prop_board.build_history_from_games` / `project_stat_moments` /
  `project_prop_line`, so they can't drift (parity pinned by test).
- **WS8 cross-book layer (2026-07-16)** — `cross_book_arb.py`:
  `find_cross_book_opportunities` (per (player, stat) with ≥2 books: line_gap,
  best over/under book, middle candidates), `fetch_two_way_lines` +
  `detect_two_way_arb` (TRUE-arb flag only when RAW
  `implied_over_A + implied_under_B < 1` in an executable direction, from REAL
  `betting_lines` odds — never from assumed -110). CLI `--min-gap` / `--arb-only`.
  Streamlit **"Cross-book"** view (KPIs incl. arb count, distinct true-arb section,
  CSV export, Player Charts jump, offseason empty-state); its book list unions
  `web_prop_cards` + `betting_lines` so real-odds-only books stay selectable.
- **WS10 tooling (2026-07-16; measurement only, no execution layer)** —
  `bet_log` table + `insert_bet_log_rows` / `settle_bet_log(fill_clv=)` (grades
  pending picks vs `game_logs`, fills `clv_delta` from closing
  `betting_line_snapshots`); `bet_slip.py` CLI = the paper-trade entry point
  (full-mode edges → min_edge/min_p/max_picks/capped-Kelly gates → CSV+JSON
  artifacts + `bet_log` insert; `--dry-run` = no side effects);
  `calibration_report.py` (per-stat reliability buckets + Brier from settled
  `predictions` or `bet_log`); `hourly_update --settle-bet-log` (default OFF).
- **Full model path** — 8 distributions; team priors in `run_single_prop`, `prop_board`,
  hourly predictions (points/assists/rebounds/pra). Rebounds default → poisson; others → normal.
- **Player Charts** — Plotly web + matplotlib Tk; line ladder, movement tab, CLV/Kelly,
  splits, ribbon, etc. Upcoming-game line selection + scrape staleness timestamps.
- **Web / billing** — Feature parity with Tk; watchlist (cookie + signed-in JSON); Stripe
  portal, admin dashboard, trial via `created_at`, Postgres subs via `SUBSCRIPTIONS_DB_URL`.
- **Ops** — `publish_db.sh`, ETL alerts, hourly scheduler, login-wall detection,
  `historical_odds_backfill.py`, FanDuel/BetRivers/ESPNBet team-line parsers (fixtures).
- **Multi-sport** — `sport` column migration; `(book, sport)` scraper resolution; MLB Stats
  API ingest + `mlb_game_logs`; FanDuel MLB team lines validated live; NFL scaffold only.

## Model accuracy & Edge Scanner (honest baseline)

**Edge Scanner ≠ arbitrage.** It ranks **model edge** (forecast vs line), not locked profit.

| Term | Meaning | Built? |
|------|---------|--------|
| Model edge | P(side) vs book implied | Yes (Edge Scanner, 3 modes) |
| Line shopping | Best over/under line across books | Yes (`cross_book_arb` + Cross-book view) |
| Middle | Over low line + under high line | Candidates flagged via `line_gap` — never "guaranteed" |
| True arb | Σ implied < 1 across legs (real odds both sides) | Detector built (`detect_two_way_arb`); **data-starved** until sportsbook odds land in `betting_lines` |

**Distributions:** engineering solid; **validated market edge still unproven.** Only
rebounds→poisson showed positive sweep ROI (+8.8%, n=482, **not significant**).
Points/assists/pra stay normal (sweep inconclusive / negative ROI when bets placed).
`--use-market-lines` needs more `betting_lines` volume. High win rates in old
real-data benchmarks used rolling mean as the line (model vs itself) — not book CLV.

**WS9 closed the modeling gap; the remaining gap is proof:**

| | `chart_mean` (quick screen) | `full` (WS9) | Still missing |
|--|--|--|--|
| μ / σ | Last-N mean / sample std | Rolling + team priors | Defense/minutes adjustments in scanner path |
| Dist | Always normal | `get_default_distribution(stat)` | Per-stat σ tuning |
| Proof | None | None yet | Settled `bet_log` + CLV + market-line backtests (WS10) |

## Known limitations

- Sparse scraped lines → Edge Scanner / Cross-book / Line Movement / CLV mostly empty
  until the Mac ETL runs in-season (offseason arb count 0 = expected).
- `betting_lines` thin → market-line validation + TRUE-arb blocked (backfill tooling exists).
- Team charts: per-game aggregates for points/assists/rebounds/pra/3pm/fgm. Only
  `points` has a posted book reference (implied team total); other stats get a
  props-derived reference (Σ players' consensus prop lines) shown only when ≥5
  players have current props — thin/offseason coverage shows an empty-state.
  `players.team` sparse, so team membership uses the `matchup` fallback.
- Scraping host = dev Mac; dev box may be Windows (`DataLoader` sqlite leak on temp DB
  teardown — scanner `full` mode is DB-direct and unaffected).
- Hourly `predictions` NaN rows for NULL-in-window stats: fixed 2026-07-21 (root-fix in
  `build_history_from_games` + `_persist_predictions` guard + `delete_nonfinite_predictions`
  cleanup). Below the valid-games floor a row is intentionally skipped, not projected.
- **Do not use Edge Scanner / bet_slip picks for real money** until WS10 Phases 1–2 pass.
