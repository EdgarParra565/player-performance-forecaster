### PrizePicks auth workflow (CDP approach — the only reliable method)
PrizePicks runs three simultaneous bot-protection layers (Cloudflare, PerimeterX, DataDome). All three tie session cookies to the browser's TLS fingerprint. Playwright's Chromium has a different TLS fingerprint from real Chrome — cookies from Chrome cannot be reused in Playwright, and no JS-side stealth patch can fix TLS-layer detection.

**Solution: route everything through a real running Chrome via CDP.**

```bash
# Step 1: Launch Chrome with remote debugging (one-time per session)
open -na "Google Chrome" --args \
    --remote-debugging-port=9222 \
    --user-data-dir=/tmp/pp-chrome-profile

# Step 2: Log in to PrizePicks in that Chrome window manually
# (complete any Cloudflare/location verification, get to the board)

# Step 3: Extract full session (cookies + localStorage JWT)
.venv/bin/python3 -m nba_model.model.web_text_ingestion \
  --connect-chrome https://app.prizepicks.com/board/nba \
  --browser-auth-state-file data/config/auth/prizepicks_state.json \
  --chrome-debug-port 9222

# Step 4: Validate session through real Chrome
.venv/bin/python3 -m nba_model.model.web_text_ingestion \
  --validate-session https://app.prizepicks.com/board/nba \
  --browser-auth-state-file data/config/auth/prizepicks_state.json \
  --chrome-debug-port 9222

# Step 5: Fetch/scrape through real Chrome (keep Chrome open while running)
.venv/bin/python3 -m nba_model.model.web_text_ingestion \
  --urls "https://app.prizepicks.com/board/nba" \
  --browser-auth-state-file data/config/auth/prizepicks_state.json \
  --chrome-debug-port 9222 \
  --force-poll
```

## 5) Known Gaps / Constraints

- Reverse-engineering quality depends on enough overlapping `betting_lines` + model predictions in the same date/player/stat windows.
- Name mismatches across sources can still reduce join coverage (e.g., punctuation/suffix variants in player names).
- Some evaluation scripts still produce empty artifacts when windows are sparse; they now fail less often, but output quality remains data-dependent.
- ETL scheduler exists (GitHub Actions), but external alerting/monitoring on failed or partial runs is still minimal.
- UnderDog anonymous access returns login-gated UI; use `--login` to save session first, then headless fetch with `--browser-auth-state-file`.
- Session persistence depends on browser fingerprint compatibility; some sites may invalidate headless sessions saved from headed logins.
- Authenticated browser ingestion requires project `.venv` + Playwright browser binaries (`.playwright-browsers`).
- CSS selector hints for book-specific waits are best-effort; DOM changes on sportsbook sites will need adapter updates.
- PrizePicks CDP fetch requires Chrome to remain open with `--remote-debugging-port=9222` — not suitable for fully unattended cron jobs without a persistent Chrome process.
- `--extract-chrome-session` (pycookiecheat) captures cookies only; `--connect-chrome` (CDP) is required to also capture localStorage JWT tokens.
- Previously: running `web_text_ingestion` and `browser_prop_parser` in unison caused the script to hang after session validation (resolved by separating fetch and parse steps).

================================================================================
TECHNICAL DEBT / KNOWN ISSUES
================================================================================

- Daily ETL runner exists and is scheduled, but external monitoring/alerting is still minimal.
- Reverse-engineering outputs are more stable with continuous mode, but remain data-coverage constrained.
- Line comparison/monthly diagnostics can still return sparse/empty artifacts when `betting_lines` and `predictions` do not overlap enough.
- Real-data benchmark/distribution sweep scripts are implemented; production defaults still need formal selection from full-run artifacts.
- UnderDog fetch requires authenticated session; use `--login` to manually save session, then `--browser-auth-state-file` for headless reuse.
- PrizePicks fetch requires real Chrome running with `--remote-debugging-port=9222`; use `--connect-chrome` to capture session and `--chrome-debug-port` for all subsequent fetches. Playwright's Chromium is blocked at the TLS layer by Cloudflare + PerimeterX + DataDome regardless of JS stealth patches.
- Session persistence across headed->headless may fail if site validates browser fingerprint (user-agent, TLS, etc).
- CSS selector wait hints for books are best-effort; DOM restructures require updating `_BOOK_WAIT_SELECTORS` map.
- Current travel/rest/injury features are heuristic; no independent validation source is integrated yet.
- Some modules still use `print()` instead of structured logging.
- DB path/config remains mostly default-arg based; centralized env config is incomplete.
- No explicit handling for player trades/team-context shifts mid-season.
